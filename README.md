# Zero Secret Server Authentication Demo

Невеликий демонстраційний прототип для бакалаврської роботи на тему:
«Методологія та інструментальні засоби розробки систем автентифікації на основі моделі Zero Secret Server Authentication».

## Призначення прототипу

Репозиторій підготовлено як чисту стартову основу для подальшого розвитку дослідницького проєкту. Поточна версія не намагається реалізувати повну систему автентифікації, а лише ілюструє базову ідею: локальна обробка секрету користувача та формування демонстраційної відповіді без реального сервера і без збереження чутливих даних у репозиторії.

## Що реалізовано зараз

- статична сторінка `index.html`, яку можна відкрити локально в браузері;
- форма з полями ідентифікатора користувача та локального секрету;
- локальна симуляція потоку автентифікації у браузері;
- використання Web Crypto API для хешування значень;
- генерація одноразового `nonce`;
- обчислення демонстраційної відповіді (`demo response`);
- покроковий журнал виконаних дій;
- чітке попередження, що це лише навчальний прототип.

## Що ще не реалізовано

- реальний сервер і серверна перевірка автентифікації;
- база даних або будь-яке постійне збереження даних;
- реєстрація користувачів;
- захист від перебору, повторного використання виклику чи аналізу атак;
- транспортний захист, керування сесіями та авторизація;
- production-ready криптографічний протокол.

## Структура репозиторію

- `index.html` — основна HTML-сторінка з інтерфейсом демонстрації.
- `styles.css` — мінімальні стилі для зручного читання та запуску локально.
- `script.js` — логіка локальної симуляції автентифікації в браузері.
- `README.md` — опис прототипу, його меж та способу запуску.
- `.gitignore` — простий список файлів, які не слід додавати до репозиторію.
- `license` — файл ліцензії проєкту.

## Як запустити локально

1. Завантажити або клонувати репозиторій.
2. Відкрити файл `index.html` у сучасному браузері.
3. Ввести ідентифікатор користувача та локальний секрет.
4. Натиснути кнопку симуляції та переглянути журнал кроків.

## Зауваження щодо безпеки

Репозиторій не повинен містити приватні ключі, реальні паролі, токени, файли `.env` або будь-які інші чутливі дані. Поточна версія призначена лише для навчальної демонстрації підходу та підготовки до подальшої розробки.

## Статус

Цей репозиторій є демонстраційним прототипом, підготовленим для подальшого розвитку в межах бакалаврського проєкту.
